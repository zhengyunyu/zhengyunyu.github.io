# Computed&Watch

# computed

computed看上去是方法，实际上是计算属性。

它会根据你所依赖的数据动态显示新的计算结果，计算结果会被缓存，computed的值在getter执行后是会缓存的，只有在它依赖的属性值改变之后，下一次获取computed的值时才会重新调用对应的getter来计算

# 特点

  1. 支持缓存，只有依赖数据发生变化，才会重新进行计算

  2. 不支持异步，当computed内有异步操作时无效，无法监听数据的变化

  3. 如果一个属性是由其他属性计算而来的，这个属性依赖其他属性，一般用computed

示例：

可以是一个方法，也可以是一个对象

<template>
  <div>
    <el-input v-model="firstText"></el-input>
    <el-input v-model="lastText"></el-input>
    <el-input v-model="mergeText1"></el-input>
    <el-input v-model="mergeText2"></el-input>
  </div>
</template>

<script>
export default {
  data() {
    return {
      firstText:'hello',
      lastText:'world',
    };
  },
  computed:{
    mergeText1(){
      return this.firstText + ' ' + this.lastText;
    },
    mergeText2:{  /* 通过mergeText2反向赋值给 firstText和lastText */
      // getter
      get() {  // 回调函数 当需要读取当前属性值是执行，根据相关数据计算并返回当前属性的值
        return `${this.firstText} ${this.lastText}`;
      },
      // setter
      set(val) {  //监视当前属性值的变化，当属性值发生变化时执行，更新相关的属性数据,val就是fullName的最新属性值
        const names = val.split(' ');
        console.log(names);
        this.firstText = names[0];
        this.lastText = names[names.length - 1];
      }
    }
  },
};
</script>

# 缓存原理

Vue`在初始化时，会调用原型上的`init`函数，将`options`传入，在函数中获取`options`上的`computed`配置，初始化`computed
// src/init.js

import initState from './initState.js'
import initComputed from './initComputed.js'

Vue.prototype._init = function (options) {
    const vm = this
    vm.$options = options
    if (options.data) {
        // 初始化数据
        initState(vm)
    }
    if (options.computed) { // 新增
        // 初始化computed
        initComputed(vm)
    }
}

initComputed

在initComputed中，需要遍历computed中的每一项，判断其是对象还是函数，然后将其方法返回给getter。

接着给每一个key创建一个Watcher，称之为computed watcher，还要传入一个lazy:true，这是为了标致这个watch是computed watch

同时要把这个key挂载到vm这个实例上，让它成为vm的属性

// src/initComputed.js

import { Dep } from "./Dep"
import Watcher from "./Watcher"

export default function initComputed(vm) {
  const computed = vm.$options.computed // 拿到computed配置
  const watchers = vm._computedWatchers = Object.create(null) // 给当前的vm挂载_computedWatchers属性，后面会用到
  // 循环computed每个属性
  for (const key in computed) {
    const userDef = computed[key]
    // 判断是函数还是对象
    const getter = typeof userDef === 'function' ? userDef : userDef.get
    // 给每一个computed创建一个computed watcher 注意{ lazy: true }
    // 然后挂载到vm._computedWatchers对象上
    watchers[key] = new Watcher(vm, getter, () => {}, { lazy: true })
    if (!(key in vm)) {
      defineComputed(vm, key, userDef)
    }
  }
}

Watcher

Watcher里面的value，会根据watcher的类型进行判断赋值，如果是computed watcher，那么一开始并不会直接去计算。

后面执行render函数的时候，lazy才会被改变，才会进行调用get方法然后进行计算

import { pushTarget, popTarget } from './Dep'

class Watcher {
  constructor(vm, exprOrFn, cb, options) {
    this.vm = vm
    if (typeof exprOrFn === 'function') {
      this.getter = exprOrFn
    }
    if (options) {
      this.lazy = !!options.lazy // 为computed 设计的
    } else {
      this.lazy = false
    }
    this.dirty = this.lazy
    this.cb = cb
    this.options = options
    this.id = wId++
    this.deps = []
    this.depsId = new Set() // dep 已经收集过相同的watcher 就不要重复收集了
    this.value = this.lazy ? undefined : this.get()
  }
  get() {
    const vm = this.vm
    pushTarget(this)
    // 执行函数
    let value = this.getter.call(vm, vm)
    popTarget()
    return value
  }
  addDep(dep) {
    let id = dep.id
    if (!this.depsId.has(id)) {
      this.depsId.add(id)
      this.deps.push(dep)
      dep.addSub(this);
    }
  }
  update(){
    if (this.lazy) {
      this.dirty = true
    } else {
      this.get()
    }
  }
  // 执行get，并且 this.dirty = false
  evaluate() {
    this.value = this.get()
    this.dirty = false
  }
  // 所有的属性收集当前的watcer
  depend() {
    let i = this.deps.length
    while(i--) {
      this.deps[i].depend()
    }
  }
}

defineComputed

defineComupted的主要作用就是让computed挂载到Vue实例上。

这里会经过判断computed是对象还是函数，然后进行不同操作。

如果是函数的话，会先获取到对应key的Watcher，接着判断dirty标识，dirty标识是computed之所以能进行缓存的关键。

dirty一开始是true，说明这时需要计算，就会调用watcher实例上的evaluate，此时才会把该watcher上的value进行计算赋值，然后把dirty设置为false，表示已经计算完成，后续不必再计算，除非依赖的属性变化了。

// src/initComputed.js


function defineComputed(vm, key, userDef) {
  let getter = null
  // 判断是函数还是对象
  if (typeof userDef === 'function') {
    getter = createComputedGetter(key)
  } else {
    getter = userDef.get
  }
  Object.defineProperty(vm, key, {
    enumerable: true,
    configurable: true,
    get: getter,
    set: function() {} // 又偷懒，先不考虑set情况哈，自己去看源码实现一番也是可以的
  })
}
// 创建computed函数
function createComputedGetter(key) {
  return function computedGetter() {
    const watcher = this._computedWatchers[key]
    if (watcher) {
      if (watcher.dirty) {// 给computed的属性添加订阅watchers
        watcher.evaluate()
      }
      // 把渲染watcher 添加到属性的订阅里面去，这很关键
      if (Dep.target) {
        watcher.depend()
      }
      return watcher.value
    }
  }
}

完整流程

  1. 初始化的时候，先对computed里面的配置进行初始化，将其挂载到实例上，同时创建一个它对应的Watcher

  2. 接着render函数执行，读取到了computed，computed触发了自身的get发放，也就是computedGetter，触发到watcher实例的方法，计算出其值赋值给value，同时设置缓存标志位为false，说明其下次使用可以直接返回value。同时，它会把此时的渲染 watcher进行一个收集，触发depend方法，该方法是为了让computed watcher所依赖的属性也能收集到渲染 watcher。

  3. 由于在计算其值的时候，会get到他对应的依赖，所以也会触发到这些依赖的get方法，依赖的get会把这时的Watcher，也就是computed watcher推入到自己的dep中。

  4. 当依赖的属性值发生变化时，会遍历自身的dep（dep中都是watcher）去通知对应的watcher派发更新，这时computed watcher就会被触发到update函数

  5. 在update中，会把缓存标志位dirty设置为true，说明其需要重新计算。

  6. 这时依赖的属性会继续执行自己dep中的watcher，这时就轮到渲染watcher了，渲染watcher一执行，就会触发computed中的get方法，因为dirty已经被改变了，所以会重新计算，这就是缓存的由来。

注意：

computed自己是没有dep的，所以这时候watcher和dep的双向收集就有了意义。

computed watcher中的deps，收集了其依赖的属性，而自己可以通过遍历这些属性的dep，让他们去收集渲染watcher，这样就可以：

<div>{{person}}</div>

computed: {
    person {
        return `名称：${this.name}`
        }
    }

这里的person是依赖于name的，但是person是没有自己的dep的（因为他是computed属性变量），而name是有的。好了，继续看，请注意，此例子html里只有person的引用没有name的引用，所以name一改变，按理说虽然person跟着变了，但是html不会重新渲染，因为name虽然有dep，有更新视图的能力，但是奈何人家html不引用他啊！person想要自己去更新视图，但他却没这个能力啊，毕竟他没有dep这个管家！这个时候computed Watcher里收集的name的dep就派上用场了，可以借助这些dep去更新视图，达到更新html里的person的效果。

# watch

watch像是一个data数据的监听回调，当依赖的data数据变化，执行回答，在方法中传入newVal和oldVal。

# 特点

  • 不支持缓存，数据变化，直接会触发相应的操作；

  • watch 支持异步操作；

  • 监听的函数接收两个参数，第一个参数是最新的值；第二个参数是输入之前的值；

  • 当一个属性发生变化时，需要执行对应的操作，一对多；

  • 监听数据必须是 data 中声明过或者父组件传递过来的 props 中的数据。当数据变化时触发其他操作，函数有两个参数： immediate：组件加载立即触发回调函数执行； deep: 深度监听；为了发现对象内部值的变化，复杂类型的数据时使用，例如：数组中的对象内容的改变，注意：监听数组的变动不需要这么做。注意：deep无法监听到数组的变动和对象的新增，参考vue数组变异,只有以响应式的方式触发才会被监听到；

# 实现

主要是在数据更新的时候,通知user watcher去执行传入的回调
