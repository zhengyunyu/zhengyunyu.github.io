Symbol

# 特性

ES6引入了一种新的原始数据类型Symbol，表示独一无二的值，最大的用法是用来定义对象的唯一属性名。

可以认为Symbol相当于一种类似String的基本数据类型，通过Symbol()生成。

  • 由于本质上是基本类型所以不能加上new

  • Symbol():

    • 一般传String为参数以通过symbol.toString()或控制台输出区分，但其参数只是表示对当前Symbol值的描述，因此相同参数的Symbol函数的返回值是不同的
    • 假如以Object作为参数，上述两种区分方式会调用该对象的toString()

  • 不能与其他类型的值进行运算或连接

  • 没有值可以显示或隐式地转为Symbol，Symbol也不能被隐式自动地转为字符串。但后者可以显示转String或Boolean，但不能转为Number

let sym1 = Symbol('My symbol');
String(sym1); // 'Symbol(My symbol)'
sym1.toString(); // 'Symbol(My symbol)'

let sym2 = Symbol();
Boolean(sym2); // true
!sym2; // false
if (sym2) {
  // ...
}
Number(sym2); // TypeError
sym2 + 2; // TypeError

# Symbol注册表

Symbol.for(key)方法会根据给定的键key，来从运行时的Symbol注册表中找到对应的Symbol，如果找到了则返回，否则会新建一个与该键相关联的Symbol，放入全局Symbol注册表中。

  • Symbol.for()和Symbol()中两种写法，都会生成新的Symbol。区别在于前者会被登记在全局环境中供搜索，后者不会。
  • Symbol.for()不会每次调用就返回一个新的Symbol类型的值，而是会先检查给定的key是否存在，不存在才会新建一个值。
let yellow = Symbol("Yellow");
let yellow1 = Symbol.for("Yellow");
yellow === yellow1;      // false
 
let yellow2 = Symbol.for("Yellow");
yellow1 === yellow2;     // true

Symbol.keyFor(sym)方法用来获取symbol注册表中某个symbol键

let yellow1 = Symbol.for("Yellow");
Symbol.keyFor(yellow1);    // "Yellow"

# 属性名

js对象的属性名有两种类型：String和Symbol，遍历对象时后者不会出现在以下方式中：

  • for..in

  • for..of

  • Object.keys()

  • Object.getOwnPropertyNames()

  • JSON.stringify()

但他也不是私有属性：

  • Object.getOwnPropertySymbols()可获取指定对象的所有Symbol属性名，该方法返回一个数组，成员是当前对象的所有用作属性名的Symbol值
  • Reflect.ownKeys()方法可以返回所有类型的键名，包括常规键名和Symbol键名
let mySymbol = Symbol();
// 第一种写法
let a = {};
a[mySymbol] = 'Hello!';
// 第二种写法
let a = {
  [mySymbol]: 'Hello!'
};
// 第三种写法
let a = {};
Object.defineProperty(a, mySymbol, { value: 'Hello!' });
// 以上写法都得到同样结果
a[mySymbol] // "Hello!"

注：Symbol()作为对象属性名时不能用 . 运算符，要用方括号。因为点运算符后面是字符串，所以取到的是字符串sy属性，不是Symbol值sy属性

let sy = Symbol("key1");
let syObject = {};
syObject[sy] = "kk";
 
syObject[sy];  // "kk"
syObject.sy;   // undefined

# 防止属性污染

为对象添加一个属性,此时可能造成属性覆盖,用Symbol作为对象属性可以保证永远不会出现同名属性,如模拟手写call

Function.prototype.myCall = function(context) {
  // 用于防止 Function.prototype.myCall() 直接调用
  if (typeof this !== 'function') return undefined;
  context = context || window;
  const fn = Symbol();
  context[fn] = this;
  const args = [...arguments].slice(1);
  const result = context[fn](...args);
  delete context[fn];
  return result;
};

# 私有属性

借助Symbol类型的不可枚举,我们可以在类中模拟私有属性,控制变量读写

const privateField = Symbol();
class myClass {
  constructor(){
    this[privateField] = 'ConardLi';
  }
  getField(){
    return this[privateField];
  }
  setField(val){
    this[privateField] = val;
  }
}

# 不重名常量

可用于消除魔术字符串,后者指的是在代码中多次出现,与代码形成强耦合的某一个具体的字符串或数组

// 字符串 basic、super 就是与业务代码无关的魔法字符
if (type === 'basic') {
  return <div>basic tab</div>;
}

if (type === 'super') {
  return <div>super tab</div>;
}

// 改造后
const tabTypes = {
  basic: Symbol(),
  super: Symbol(),
};

if (type === tabTypes.basic) {
  return <div>basic tab</div>;
}

if (type === tabTypes.super) {
  return <div>super tab</div>;
}
