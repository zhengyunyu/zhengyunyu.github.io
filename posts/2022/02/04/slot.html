Slot

# 什么是插槽

Vue实现了一套内容分发的API，将slot元素作为承载分发内容的出口，具体来说，slot就是可以让你在组件内添加内容的‘空间’。

# Vue插槽slot的基本使用

# 单个插槽|匿名插槽

//子组件 ： (假设名为：child)
<template>
  <div class= 'child'>
      
  </div>
</template>

//父组件：（引用子组件 child）
<template>
  <div class= 'app'>
     <child> 
        RunHio
     </child>
  </div>
</template>

如果想在父组件中添加内容“RunHio”，文本是不会被在页面上被渲染的。

如果想要添加的内容能够显示，在子组件内添加slot即可。

//子组件 ： (假设名为：child)
<template>
  <div class= 'child'>
      <slot></slot>
  </div>
</template>

子组件的slot中也可以设置一个默认值，当父组件内没有添加内容时，slot就会展示默认值

//子组件 ： (假设名为：child)
<template>
  <div class='child'>
      <slot>这就是默认值</slot>
  </div>
</template>

# 具名插槽

有时候，子组件内的slot不止一个，想要在父组件中精确定位到插槽位置，就要使用具名插槽，给slot命名

<template>
  <div class='child'>
    <slot name='one'>one</slot>
		<slot name='two'>two</slot>
	</div>
</template>

父组件可以通过slot='name'(Vue2旧语法)，v-slot或#name(Vue3新语法)的方式添加内容

//父组件：（引用子组件 child）
<template>
  <div class= 'app'>
     <child> 
        <template v-slot:one> 这是插入到one插槽的内容 </template>
        <template v-slot:two> 这是插入到two插槽的内容 </template>
     </child>
  </div>
</template>

# 作用域插槽

即父组件在子组件的slot处使用子组件的data。

正常情况下，如果父组件想直接通过插槽去使用子组件的内容是做不到的，因为父级模板里的所有内容都是在父级作用域中编译的，子模板里的所有内容都是在子作用域中编译的

// 子组件 ： (假设名为：child)
<template>
  <div class= 'child'>
       <slot></slot>
  </div>
</template>

new Vue({
  el:'child',
  data:{
    child:'子组件'
  }
})

// 父组件：（引用子组件 child）

<template>
  <div class= 'app'>
     <child> {{ child }}</child>
  </div>
</template>

不过利用作用域插槽可以

这里是Vue3的写法，可以将具名和作用域融合到一起写

//子组件 ： (假设名为：child)
<template>
  <div class= 'child'>
      <slot name= 'one' :value1='child1'> 这就是默认值1</slot>    //绑定child1的数据
      <slot :value2='child2'> 这就是默认值2 </slot>  //绑定child2的数据，这里我没有命名slot
  </div>           
</template>

new Vue({
  el:'child',
  data:{
    child1:'数据1',
    child2:'数据2'
  }
})

//父组件：（引用子组件 child）
<template>
  <div class='app'>
     <child> 
        <template v-slot:one='slotone'>  
           {{ slotone.value1 }}    // 通过v-slot的语法 将子组件的value1值赋值给slotone 
        </template>
        <template v-slot:default='slotde'> 
           {{ slotde.value2 }}  // 同上，由于子组件没有给slot命名，默认值就为default
        </template>
     </child>
  </div>
</template>

# Slot插槽是怎么“插的”

# 普通插槽

//子组件 ： (假设名为：child)
<template>
  <div class='child'>
      我在子组件里面
      <slot></slot>
      <slot name="one"></slot>
  </div>
</template>

//父组件：（引用子组件 child）
<template>
  <div class= 'app'>
     <child> 
        这是插入到默认插槽的内容 {{parent}}
        <template v-slot:"one"> 这是插入到one插槽的内容 {{parent}}</template>
     </child>
  </div>
</template>

new Vue({
  el:'.app',
  data:{
    parent:'父组件的值'
  }
})
  1. 首先，父组件先解析，把child当作子元素处理，把插槽当作child的子元素处理，同时在父组件的作用域内得到parent变量的值。
{    
  tag: "div",    
   children: [{        
      tag: "child",        
      children: ['这是插入到默认插槽的内容 父组件的值', 
                  '这是插入到one插槽的内容 父组件的值']
  }]
}
  1. 子组件解析，slot作为一个占位符，会被解析成一个函数
{    
    tag: "div",    
    children: [
        '我在子组件里面',
        _t('default'), // 匿名插槽，默认名称为default
        _t('one') // 具名插槽，名称为one
    ]
}
  1. _t函数传入插槽的名字，这个函数的作用是把第一步解析到的插槽节点拿到，然后返回解析后的节点
{    
    tag: "div",    
    children: ['我在子组件里面', 
                '这是插入到默认插槽的内容 父组件的值', 
                '这是插入到one插槽的内容 父组件的值']
}

# 作用域插槽

//子组件 ： (假设名为：child)
<template>
  <div class= 'child'>
      <slot :value1='child1' :value2='child1'></slot>
      <slot name='one' :value1='child2' :value2='child2'></slot>
  </div>           
</template>

new Vue({
  el:'child',
  data:{
    child1: '子数据1',
    child2: '子数据2'
  }
})

//父组件：（引用子组件 child）
<template>
  <div class='app'>
     <child> 
         <template v-slot:default='slotde'> 
            插入默认 slot 中{{ slotde.value1 }}{{ slotde.value2 }}
        </template>
        <template v-slot:one='slotone'> 
            插入one slot 中{{ slotone.value1 }}{{ slotone.value2 }}
        </template>
     </child>
  </div>
</template>
  1. 同样是父节点先解析，当遇到作用域插槽时，会将这个插槽封装成一个函数保存在child的子元素下面
{    
 tag: "div",    
  children: [{        
     tag: "child"
     scopeSlots:{            
         default (data) { // 记住这个data参数               
             return ['插入one slot 中插入默认 slot 中' + data.value1 + data.value2]
         },
         one (data) { // 记住这个data参数             
             return ['插入one slot 中' + data.value1 + data.value2]
         }
     }
    }]
}
  1. 接着子组件解析，会把子组件对应的插槽数据包装成一个对象，传入_t函数中
{    
 tag: "div",    
   children: [
     '我在子组件里面',
      _t('default',{value1: '子数据1', value2: '子数据1'}),
      _t('one',{value1: '子数据2', value2: '子数据2'})
      
    ]
}
  1. 接着_t内部执行，包装后的对象会被当作data参数 传入scopeSlots中对应的各个函数
{    
  tag: "div",    
   children: [
      '我在子组件里面', 
      '插入默认 slot 中 子数据1 子数据1',
      '插入one slot 中 子数据2 子数据2'
   ]
}
